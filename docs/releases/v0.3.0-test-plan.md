# AIGTD Reminders v0.3.0 测试文档

版本：`0.3.0 (13)`
用途：手动验收 0.3.0 候选版本，确认现有单次 Action 链路更稳定、更真实、更可诊断。

## 测试目标

`0.3.0` 不是长期记忆、多工具循环或 AI 自动深度规划版本。本版本主要验证：

- AI 回复不会暴露 JSON、结构化 action 或内部字段。
- 任务执行结果真实，成功才说成功，失败必须说明失败。
- 创建、修改、移动、完成、删除、重排等单次操作可稳定落到 Apple Reminders。
- “刚才这个”“它”“那条任务”等近期上下文引用可用。
- 批量重排能够真实写入 Reminders，并展示总数、成功数、失败数。
- 普通聊天不会误触发任务操作。
- Reminders 页面和 Agent 诊断页符合 0.3.0 的交互要求。

## 测试前准备

- 安装 `0.3.0 (13)` 或更高构建号的 App。
- 打开系统提醒事项权限。
- Apple Reminders 中建议准备这些清单：`收集箱`、`项目`、`下一步行动`、`等待中`、`也许以后`。
- 建议准备几条过期任务、无时间任务、同名或相似标题任务，方便验证失败和歧义场景。
- 测试时记录每一步是否通过，以及异常截图。

## 核心测试用例

### 1. 创建任务

操作：

```text
明天上午 9 点提醒我开会
```

通过标准：

- AI 回复已创建。
- 出现任务执行卡片。
- Apple Reminders 中能看到任务。
- 任务标题、时间、清单正确。

### 2. 基于上下文修改刚创建的任务

操作：

```text
改到后天上午 10 点
```

通过标准：

- App 能识别“刚创建的任务”。
- 不会新建重复任务。
- 原任务时间被更新为后天上午 10 点。
- 回复和卡片显示修改成功。

### 3. 删除上下文任务

操作：

```text
删除刚才这个任务
```

通过标准：

- 删除的是刚才那条任务。
- 卡片显示删除成功。
- Apple Reminders 中不再出现该任务。
- 不会删除其他任务。

### 4. 连续上下文操作

操作：

```text
新建任务：周五交报告
把它放到项目清单
再改到下周一
```

通过标准：

- 后两句能理解“它”指刚创建的任务。
- 不会反复询问是哪条任务。
- 不会新建重复任务。
- 最终任务在 `项目` 清单，时间为下周一。

### 5. 批量重排

操作前准备：

- 在 `收集箱` 中准备多条未完成任务。
- 其中最好包含过期任务和无时间任务。

操作：

```text
帮我把收集箱里未完成任务重新安排到后面两周内
```

通过标准：

- AI 先给出重排说明或处理结果。
- 任务被逐条更新时间。
- 结果卡片显示总数、成功数、失败数。
- 如果有失败项，必须明确显示失败项或失败原因。
- 不能只回复“我帮你安排了”但没有真实写入 Reminders。

### 6. 部分失败场景

操作方式任选其一：

```text
把不存在的任务改到明天
```

或临时关闭提醒事项权限后执行创建、修改、重排。

通过标准：

- 不能假装成功。
- 回复和卡片都要显示失败。
- 失败原因清楚，例如找不到任务、权限不足、无法解析目标、写入失败。
- 不应该产生错误的新任务。

### 7. 删除或移动歧义

操作前准备：

- 准备两个同名或相似任务，例如 `开会`、`开会准备材料`。

操作：

```text
删除开会
```

通过标准：

- 如果无法唯一确定目标，应该追问或提示歧义。
- 不能随便删除其中一条。
- 卡片不能显示误导性的成功。

### 8. 普通聊天不执行任务

操作：

```text
我今天有点累，不想整理任务
```

通过标准：

- 只做自然语言回应。
- 不创建任务。
- 不修改任务。
- 不弹出任务执行卡片。

### 9. Reminders 页面同步和清单展示

操作：

- 进入 Reminders 页面。
- 等待同步或下拉刷新。
- 对照 Apple Reminders 的清单顺序和内容。

通过标准：

- 顶部显示 `最新同步 x 秒前`。
- 清单顺序和系统提醒事项一致。
- 没有任务的清单也显示出来。
- 页面不再出现旧的黑色风格模块。
- 切换页面后数据不丢失。

### 10. Agent 诊断页

操作：

- 执行几次聊天和任务操作。
- 进入 Agent 页面查看诊断记录。

通过标准：

- 能看到最近请求的阶段、状态和摘要。
- 默认不泄露完整用户原文。
- 默认不泄露 API Key、Bearer Token、access key 等敏感内容。
- 失败记录能辅助判断问题发生在识别、远端请求、解析、执行或刷新阶段。

## 通过判定

可以认为 `0.3.0` 手动验收通过，需要同时满足：

- 创建、修改、删除、移动、完成等基础操作没有明显错误。
- 上下文引用能稳定处理最近一次任务。
- 批量重排真实写入 Reminders，并正确展示成功和失败统计。
- 权限关闭、任务不存在、同名歧义等场景不会假成功。
- 普通聊天不会误触发任务操作。
- Reminders 页面清单顺序、空清单展示、同步文案符合预期。
- Agent 诊断页有帮助且不泄露敏感信息。
- App 不崩溃、不明显卡死，Chat、Reminders、Agent 三个页面切换正常。

## 不通过记录格式

建议每个问题按下面格式记录：

```text
场景：
输入：
预期：
实际：
截图：
是否可复现：
备注：
```
测试结论：
用例 2 失败，单任务改期被误判为批量重排。
用例 4 失败，plan_reschedule 方案可生成但无法执行。
用例 7 失败：存在两条同名任务时，App 没有澄清，直接删除了明天下午那条。这属于高风险问题，因为当前匹配策略会擅自选择一个候选任务。
普通聊天未真实执行：通过。
普通聊天错误闪现执行卡片：失败。
卡片突然消失的过渡体验：失败。
同步文案：失败，显示成“最新同步 0 秒后”，正确应为“最新同步 0 秒前”或“刚刚同步”。
