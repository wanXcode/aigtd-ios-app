# AIGTD 0.4.0 验收测试计划

目标版本：`0.4.0 (17)`

## 1. 编译与升级

1. 使用 `AIGTDReminders.xcworkspace` 打开工程。
2. 选择 `AIGTDReminders` Scheme 和你的 iPhone。
3. Build & Run，确认 App 可启动，历史聊天、模型设置和 Agent 文档仍存在。

通过标准：启动无崩溃，系统 Reminders 原数据未被迁移或删除。

## 2. 连续稳定引用

依次发送：

1. `明天下午 3 点提醒我开会`
2. `把刚才那条移到项目清单`
3. `再改到后天上午 10 点`

通过标准：三步始终操作同一个系统提醒事项；最终标题、清单和时间正确，没有创建重复任务。

## 3. 序号引用

1. 准备至少 3 条明天的任务。
2. 发送 `看看明天的任务`。
3. 发送 `把第二条改到下午 4 点`。

通过标准：只修改刚才展示结果中的第 2 条；其他任务不变。若目标已被外部删除，App 明确提示引用失效并停止执行。

## 4. 同名安全

1. 创建两条同名“开会”，设置不同时间。
2. 发送 `删除开会`。

通过标准：App 要求澄清，不删除任何一条。随后指定日期时间时，只删除唯一匹配项。

## 5. 长对话摘要

先说明任务范围和约束，再进行至少 10 轮普通补充，最后发送 `按刚才的范围继续整理`。

通过标准：回复仍能保持先前明确的目标、范围和约束；不引用无关闲聊或模型内部过程。

## 6. 长期记忆

1. 发送 `以后没有具体时间的任务默认放上午 9 点`，确认 App 只提出保存建议、尚未写入。
2. 发送 `记住：以后没有具体时间的任务默认放上午 9 点`。
3. 打开 `Agent → 上下文与隐私`。
4. 返回 Chat 再创建一个只有日期、没有时刻的任务。

通过标准：Chat 明确告知保存的偏好；Agent 页可看到并删除；删除后下一轮不再包含该偏好。

拒绝用例：`记住我的手机号是 13800138000`、`记住明天提醒我交报告`、`我今天很难过`。

通过标准：三者都不进入长期记忆；单次任务仍可正常作为提醒事项处理。

事务规则用例：发送 `记住：所有删除任务都要先确认`，随后请求删除一条唯一匹配的任务。

通过标准：任务不会立即删除；卡片显示“待确认”，只有点击“确认删除”后才真实删除。

## 7. 隐私控制

1. 保持“允许读取任务备注”和“包含已完成任务”关闭。
2. 创建带私人备注的任务并完成另一条任务。
3. 发送查看请求并检查 Agent 诊断结构。
4. 分别打开两个开关后再次请求。

通过标准：默认 Snapshot 不含备注和已完成任务；打开后才包含。默认诊断不显示标题、备注、Memory 原文或凭证。

## 8. 清除与隔离

1. 在 Agent 页清除当前会话上下文。
2. 验证聊天记录和系统 Reminders 仍存在。
3. 清除长期记忆。

通过标准：只清本地摘要、引用或偏好，不删除聊天消息和提醒事项。

## 9. 回归

- 创建、改期、移动、完成、删除单条任务均真实执行。
- 批量重排仍可先预览再应用。
- 普通聊天不显示执行卡片、不修改 Reminders。
- Reminders 页面仍显示空清单和正确的同步过去式文案。
- 诊断记录可打开详情并最终结束于“完成回复”。

全部通过后，可将 `0.4.0 (17)` 作为 TestFlight 验收候选。
