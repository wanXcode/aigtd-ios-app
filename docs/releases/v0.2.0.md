# AIGTD Reminders v0.2.0 发布说明

发布日期：2026-04-13

## 版本目标

这一版聚焦两件事：

1. 让远端大模型对话链路稳定可用（以 `responses` 为主）。
2. 把 Chat 输入区交互打磨到接近日常 IM 使用习惯（键盘、语音、滚动、状态切换）。

## 主要更新

### 1) 远端模型链路稳定性

- 完整梳理并兼容 OpenAI-compatible 的 `responses` 接口返回结构。
- 优化流式回复显示逻辑，避免“连通但无法解析”的常见场景。
- 增加远端响应调试记录能力，便于快速定位接口层问题。

### 2) Chat 输入区交互升级

- 输入框支持更稳定的多行增长与滚动。
- 键盘弹出/收起与消息列表滚动联动优化，减少遮挡和大段空白。
- 点击/滑动内容区可收起键盘，回到底部行为更自然。
- 发送后文本立即清空，发送按钮状态反馈更直接。

### 3) 语音输入（豆包 ASR）

- 接入豆包官方 WebSocket 协议链路（`/api/v2/asr`）。
- 支持语音转文字的实时更新与结束后整理。
- 支持从语音状态切换到键盘输入（键盘接管）。
- 修复多类语音状态卡住、停止不及时、尾部状态残留等问题。

### 4) Reminders 侧联动

- 改进了提醒列表读取与刷新一致性。
- 任务创建/完成等操作链路在 Chat 与 Reminders 之间同步更稳定。

## 文档与工程

- 新增并整理了 Chat 输入、语音交互相关 PRD 与验收文档。
- 项目当前通过 CocoaPods 工作区编译，依赖管理流程更清晰。

## 安全说明

- 本版本已去除代码中的硬编码模型/语音凭证默认值。
- API 凭证应由用户在 App 设置中填写并本地保存，不进入仓库。

## 回归建议（真机）

建议重点回归以下场景：

1. `responses` 模式下，多轮连续对话（含长回复）。
2. 键盘输入、语音输入、键盘接管语音三种切换路径。
3. 新建提醒、完成提醒后在系统 Reminders 与 App 内一致性。
4. 键盘收起后消息区是否始终贴底且无异常空白。
