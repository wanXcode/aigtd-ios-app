# AIGTD Reminders v0.3.0 发布说明

状态：Release Candidate
当前候选构建：`0.3.0 (14)`
版本主题：稳定基线

## 版本定位

`0.3.0` 是 AIGTD 智能化架构升级前的稳定基线版本。本版本聚焦现有单 Action 链路的真实性、可测试性和可诊断性，不引入多工具循环、长期记忆或 AI 主导的智能排期。

本文是候选发布说明。核心真机测试已通过，最终交付状态以 TestFlight 上传和处理结果为准。

## 候选更新

### 重排任务时间

- 用户提出重排要求后，先生成确定性的时间方案，不立即修改 Apple Reminders。
- 规划卡片展示范围、任务和目标时间，由用户点击“应用这个方案”后执行。
- 规划态和执行态使用同一份保存的方案，避免展示内容与实际修改不一致。
- 当前规划器按既有任务顺序，在指定时间窗口内分散安排；本版本不将其描述为 AI 智能排期。

### Action 执行真实性

- 创建、建清单、移动、完成、删除和应用重排以本地执行结果作为最终状态依据。
- 写操作执行前不使用“已经完成”类成功表达。
- 目标是统一 `pending`、`success`、`partial` 和 `failed` 状态，并避免界面重建或重复点击造成重复执行。

### 工程与测试基线

- 项目营销版本和构建号统一为 `0.3.0 (14)`。
- `project.yml` 继续作为 XcodeGen 工程配置的主要来源。
- 新增 `AIGTDRemindersTests` 单元测试 target，可通过 CocoaPods workspace 离线运行。
- 自动测试不得访问网络、API Key 或用户真实 Reminders 数据。

### 诊断与隐私

- 计划为每次请求建立可串联模型解析、本地 fallback、Action 执行和 Reminders 刷新的分阶段诊断记录。
- 默认诊断内容只保存脱敏摘要，不保存 API Key、Authorization Header、语音凭证、完整任务备注或完整聊天原文。
- 完整调试内容必须由用户明确开启，并提供保留上限和立即清除能力。

## 开发中

以下内容是 `0.3.0` 的发布门槛，目前不能据此认定功能已经完成：

- 重排逐项执行结果和部分成功状态。
- 所有写入 Action 的假成功与重复执行审计。
- 带 `traceID` 的分阶段诊断、脱敏和清理机制。
- 至少 100 条中文对话的离线评测集。
- 核心日期解析、任务匹配、方案编解码和状态归并测试。
- Debug、Release、模拟器、真机和 TestFlight 完整验收。

## 本版本不包含

- 完整 `AgentContextSnapshot` 和长期会话记忆。
- 多 `operations` 协议和多轮工具调用。
- AI 主导的智能排期算法。
- 纯文字“就按这个来”确认方案。
- 通用撤销系统。
- Chat 或 Reminders 页面的整体视觉重做。

## 发布前验收

- Debug 和 Release 均使用 `0.3.0 (5)` 或更高构建号并通过构建。
- 所有离线单元测试和对话评测通过既定门槛。
- 规划态不修改 Reminders，确认后严格执行已展示方案。
- 全部成功、部分成功和全部失败均显示真实结果。
- 权限关闭、任务不存在、同名歧义和 EventKit 写入失败均不显示成功。
- 诊断数据中不包含模型或语音凭证。
- 真机完成创建、查看、移动、完成、删除、重排、fallback 和生命周期重复执行回归。
