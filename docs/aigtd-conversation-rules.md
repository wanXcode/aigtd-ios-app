# AIGTD 对话规则

## 一、主原则

- 聊天先像人，再像系统
- 真正执行成功时才展示执行回执
- 不是每条消息都要变成任务
- 但明确的任务录入语句必须直接落任务

## 二、输入分流

### 1. 建任务

命中以下表达时，优先建任务：

- `提醒我`
- `新建任务`
- `创建任务`
- `记一个任务`
- `记个任务`
- `记个待办`
- `帮我记一个任务`
- `帮我记个待办`

示例：

- `帮我记一个任务：测试 ios 端`
- `帮我记个待办：整理报销`
- `明天提醒我给张闯回信`

### 2. 看任务

命中以下意图时，优先查看提醒事项：

- `今天我还有什么事`
- `明天我有什么`
- `看看我现在有哪些事`
- `看看收集箱里有什么`

输出方式：

- 直接自然回复
- 不显示结果卡

### 3. 试探 / 打招呼

命中以下表达时，视为普通对话：

- `测试一下这个系统`
- `试一下`
- `你在吗`
- `hello`

输出方式：

- 给一句简短自然回复
- 不创建任务
- 不显示结果卡

## 三、回复形式

### 建任务成功

应包含：

- 已经记好了
- 放到哪里
- 时间是否带上

示例：

- `记好了，哥哥。这条我帮你建进提醒事项了，放到明天。`

### 查看任务

应包含：

- 先结论
- 再点 2 到 3 条最重要的任务

示例：

- `哥哥，你今天主要还有：给张闯回信、整理报销。`

### 试探句

应包含：

- 在
- 可以做什么

示例：

- `我在，哥哥。你直接告诉我要记什么、改什么，或者问我今天还有什么事就行。`

## 四、结果卡规则

### 要显示

- 建任务成功
- 建清单成功
- 移动任务成功
- 完成任务成功

### 不显示

- 普通试探句
- 查看类回复
- 仅接住一句话但未执行任何系统动作

## 五、不要在 UI 上暴露的内容

以下内容只保留在内部，不直接展示给用户：

- intent 名称
- category 字段
- bucket 字段
- mapping rule id
- “已记录输入”这类系统味太重的文案

## 六、后续扩展

后续接入豆包语音识别 API 时，优先保证：

- 识别后的文本尽量原汁原味进入 AIGTD
- AIGTD 继续按本规则分流
- 不因为是语音输入就额外增加解释性回复
